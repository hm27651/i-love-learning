# 关键设计决定

## 学习项目是内容与进度边界

科目、计划、练习、模拟、实践任务和准备度都归属于一个学习项目。每道题只归属一个科目和知识点；跨科目使用时复制为独立题目，避免不同学习目标的进度和统计互相污染。

## 先迁移内容，再删除知识节点

包含内容的科目、章节和知识点不能级联清空。必须先在当前项目内迁移内容，或明确合并同名节点，再删除空节点。迁移时保留题目 ID、学习进度和历史试卷，降低误删整棵知识树的风险。

## 学习内容与学习记录分离

题目、知识树和导入原件属于学习内容；答题、掌握度、复习日期和模拟成绩属于学习记录。清理学习记录不得删除内容，并且必须先创建数据快照，再根据剩余答题重建掌握度。

## 单机 SQLite 串行执行后台写任务

软件继续使用单机 SQLite，不引入外部队列或数据库服务。网页请求可以并发读取，导入和导出共用一个串行后台执行器，并使用 WAL 和统一等待时间降低锁冲突，保持个人局域网部署轻量。
